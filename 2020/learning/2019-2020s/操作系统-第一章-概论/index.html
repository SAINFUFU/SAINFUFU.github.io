操作系统-第一章-概论

2020-05-20
学习-OS

目标和作用

目标

  1. 方便性（对用户）
  2. 有效性（提高资源利用率、提高系统的吞吐量）
  3. 可扩充性（无结构->模块化->层次化->微内核结构能方便地增添新的功能和模块）
  4. 开放性（更好的兼容，遵循世界标准规范OSI）

方便性和有效性是两个最重要的目标

作用

  1. 用户和计算机硬件系统之间的接口
  2. 计算机系统资源的管理者（处理机、存储器、I/O设备、文件）
  3. 实现对计算机资源的抽象

推动操作系统发展的主要动力

  1. 不断提高计算机资源利用率
  2. 方便用户
  3. 器件的不断更新迭代
  4. 计算机体系结构的不断发展
  5. 不断提出新的应用需求

发展过程

未配置操作系统的计算机系统

  1. 人工操作方式
    利用率低，出现人机矛盾
    CPU和I/O设备之间速度不匹配
  2. 脱机输入/输出方式

单道批处理系统

  1. 处理过程
  2. 缺点
    系统中的资源得不到充分的利用

多道批处理程序

  1. 基本概念
    为了提高资源的利用率和系统的吞吐量，将提交的作业先存放在外存中，排成后备队列，由作业调度程序调度。
  2. 优缺点
    (1)资源利用率高
    (2)系统吞吐量大
    (3)平均周转时间长
    (4)无交互能力
  3. 需要解决的问题

分时系统

  1. 引入
    满足用户对人机交互的需求
    需求：人机交互、共享主机
  2. 实现的关键问题
    （1）及时接收：多路卡-分时多路复用
    （2）及时处理
    作业直接进入内存
    采用轮转运行方式
  3. 特征
    （1）多路性
    （2）独立性
    （3）及时性
    （4）交互性

实时系统

做出实时或及时的反应

  1. 实时系统的类型
    （1）工业（武器）控制系统
    （2）信息查询系统
    （3）多媒体系统
    （4）嵌入式系统
  2. 实时任务的类型
    （1）周期性实时任务和非周期性实时任务
    （2）硬实时任务和软实时任务（是否必须满足对截止时间的要求）
  3. 实时系统与分时系统特征的比较
    （1）多路性
    （2）独立性
    （3）及时性
    （4）交互性
    （5）可靠性

微机操作系统的发展

  1. 单用户单任务操作系统：CP/M,MS-DOS
  2. 单用户多任务操作系统：Windows
  3. 多用户多任务操作系统：Unix OS（Solaris OS，Linux OS）

操作系统的基本特性

并发

使操作系统有效地提高系统中的资源利用率，增加系统的吞吐量

  1. 并行与并发
    并发性：两个或多个事件同一时间间隔发生（宏观上）
    并行性：两个或多个事件同一时刻发生
  2. 引入进程
    进程是系统中能独立运行并作为资源分配的基本单位，是能独立运行的活动实体。
    进程和并发是现代操作系统中最重要的基本概念，和操作系统运行的基础

共享

资源复用，供内存多个并发执行的进程共同使用，限定了时间和地点。
实现资源共享的方式：

  1. 互斥共享方式
  2. 同时访问方式

并发和共享是多用户（多任务）OS中最基本的特征。互为存在条件。

虚拟

  1. 时分复用技术（利用处理机的空闲时间运行其他程序，提高处理机的利用率）
    （1）虚拟处理机技术
    （2）虚拟设备技术
  2. 空分复用技术（利用存储器的空闲时间分区域存放和运行其他的多道程序，提高内存的利用率）

异步

进程是以人们不可预知的速度向前推进的。

操作系统的主要功能

处理机管理功能

  1. 进程控制（创建、撤销进程，控制进程在运行过程中的状态转换）
  2. 进程同步（信号量机制）
  3. 进程通信（交换信息）
  4. 调度（作业调度和进程调度）

存储器管理功能

  1. 内存分配
    （1）静态分配
    （2）动态分配
  2. 内存保护
  3. 地址映射
  4. 内存扩充（虚拟存储技术）
    （1）请求调入功能
    （2）置换功能

设备管理功能

(1)完成用户进程提出的I/O请求,为用户进程分配所需的I/O设备,并完成指定的I/O操作。
(2)提高CPU和I/O设备的利用率，提高I/O速度，方便用户使用I/O设备。

  1. 缓冲管理
  2. 设备分配
  3. 设备处理

CPU向设备控制器发出I/O命令，要求它完成指定的I/O操作；CPU接收从控制器发来的中断请求，并给予迅速的响应和相应的处理。

文件管理功能

  1. 文件存储空间的管理
  2. 目录管理
  3. 文件的读/写管理和保护

操作系统与用户之间的接口

  1. 用户接口
    （1）联机用户接口
    （2）脱机用户接口
    （3）图形用户接口
  2. 程序接口

现代操作系统的新功能

    （2）密码技术
    （3）访问控制技术

  2. 网络的功能和服务
    （1）网络通信
    （2）资源管理
    （3）应用互操作
  3. 支持多媒体

    （2）实时调度
    （3）多媒体文件的存储

微内核OS结构

  1. 基本概念
    （1）足够小的内核
    （2）基于客户/服务器模式
    （3）应用“机制与策略分离”原理
    （4）采用面向对象技术
  2. 基本功能
    （1）进程（线程）管理
    （2）低级存储器管理
    （3）中断和陷入处理
  3. 优点
    （1）提高了系统的可扩展性
    （2）增强了系统的可靠性
    （3）可移植性强
    （4）提供对分布式系统的支持
    （5）融入了面向对象技术
  4. 存在的问题
    运行效率有所降低
